Water Level Monitoring System – C Firmware Project

Overview

This simple firmware application was written in C to simulate water level monitoring in a tank. The system takes a numeric input (0–100) and categorizes it into three levels: LOW, OPTIMAL, and OVERFLOW. Based on the level, the output simulates LED indicators and pump status — mimicking embedded systems used in irrigation and manufacturing.

System Diagram

A simplified schematic shows how the firmware logic interacts with sensors and actuators in a water tank setup. Inputs (sensor data) drive logic conditions to control outputs like LEDs and the pump motor.

How the C Code Works

The code is organized with a main loop that collects user input and validates it. Then a helper function evaluates the level and prints simulated output for:

  • Water level status (LOW, OPTIMAL, OVERFLOW)
  • LED indicator (RED, GREEN, YELLOW)
  • Pump status (ON/OFF)

Example Outputs

The firmware was tested using various input levels to validate its logic and ensure real-time feedback using console-based simulation.
