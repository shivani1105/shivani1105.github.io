The Problem: Chaos in Collaboration

While collaborating via email, we frequently encountered issues such as multiple file versions, lost edits, and accidental overwrites. Coordination was time-consuming and fragile, and concurrent edits often led to code breakages, disrupting website functionality.

The Solution: Using GitHub Workflows

We transitioned from messy email threads to structured GitHub branches and commits. This eliminated version confusion and enabled smooth team collaboration.

Before GitHub, conflicting file versions often led to broken files or overwritten work. Git workflows helped us isolate, resolve, and track changes safely.

Our GitHub setup introduced a clear structure with branches like `analysis-branch`, `report-branch`, and pull requests. Everyone worked independently without overwriting each other.
