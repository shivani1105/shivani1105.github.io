The Problem: Bridging Opportunity Gaps

URM (Underrepresented Minority) is a platform that helps candidates connect with academia, recruiters, and DEI officers for PhD, postdoctoral, and faculty positions. There was a need for a centralized digital solution where users could register, log in, and create profiles for discovery and recruitment.

The Solution: URM Academic Application

My contribution: Created role-based dashboards, streamlined profile forms, and supported API integration.

Visual representation of how the control flows through the system

The DEI dashboard dynamically displays insights about candidate distribution, job listings, and roles filled.

Users are redirected to different dashboards after registering with roles: Candidate, Recruiter, DEI Officer, or Academia.

The application was designed with accessibility and role-specific interaction in mind. Each role (Candidate, Recruiter, Academia, DEI Officer) experiences a tailored dashboard post-login.

The DEI dashboard aggregates key statistics to assist officers in identifying gaps and optimizing hiring strategies. API integration enables dynamic data management and user routing between components.
